# 通用流处理任务管理平台 — 技术方案

## 1. 技术路线总览

### 架构策略：Python 统一后端 + C++ 关键路径加速

```
┌──────────────────────────────────────────────────────┐
│                    前端                               │
│       React 18 + React Flow + Ant Design 5           │
│              TypeScript / Zustand                     │
├──────────────────────────────────────────────────────┤
│                  REST API (:8080)                      │
├──────────────────────────────────────────────────────┤
│              Python FastAPI 后端                       │
│                                                       │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────┐   │
│  │ api/        │  │ engine/       │  │ cpp/       │   │
│  │ 用户 CRUD   │  │ DAG 执行器    │  │ pybind11   │   │
│  │ 控件 CRUD   │──│ 拓扑排序      │──│ C++ 加速   │   │
│  │ 作业 CRUD   │  │ 作业调度      │  │ 字段拼接   │   │
│  │ 上线/下线   │  │ 控件注册表    │  │ CSV 分块   │   │
│  └─────────────┘  └──────────────┘  └───────────┘   │
│                                                       │
├──────────────────────────────────────────────────────┤
│              MySQL 8 + Redis                          │
└──────────────────────────────────────────────────────┘
```

### 选型依据

| 层面 | 技术栈 | 选择理由 |
|------|--------|---------|
| 前端 | React 18 + React Flow | React Flow 拖拉拽生态最成熟；TypeScript 保障协作质量 |
| 后端 | Python FastAPI | 开发效率极高；API 与引擎同进程，零跨语言开销 |
| 加速 | C++ pybind11 | 关键路径（字段拼接、大文件分块）C++ 提速 5-10× |
| ORM | SQLAlchemy 2.0 (async) | 全异步，与 FastAPI 完美配合 |
| 数据处理 | Polars (Rust 实现) | 比 pandas 快 5-30×，内存效率高 |
| 存储 | MySQL 8 + Redis | 业务数据持久化 + 任务队列 |
| 构建 | Vite (前端) + pip (后端) | 极快 |

### 2.3 作业编辑器核心交互

#### 控件面板
- 按三类分组展示控件：输入 / 处理 / 输出
- 控件卡片支持拖拽到画布
- 未来支持搜索/过滤

#### 画布区域
- React Flow 实现 DAG 可视化
- 输入控件：左侧无接口，右侧有输出接口
- 处理控件：左侧输入接口，右侧输出接口
- 输出控件：左侧输入接口，右侧无接口
- 支持：缩放 / 平移 / 撤销 / 删除

#### 配置面板
- 选中节点后，根据控件类型动态渲染配置表单
- 配置项示例：
  - CSV 输入：文件路径、编码、分隔符
  - 字段拼接：A 字段、B 字段、C 字段名
  - MySQL 输出：目标表名

#### 作业操作
- 保存（存为草稿）
- 上线（提交后端，开始执行）
- 下线（停止执行）

### 3.4 控件体系设计

#### 抽象基类

```python
class BaseComponent(ABC):
    component_type: str        # "input" | "process" | "output"
    component_name: str        # 唯一标识, 如 "csv-input"
    display_name: str          # 显示名称, 如 "CSV 输入"

    async def execute(config, inputs) -> dict: ...
    def get_config_schema() -> dict: ...
```

#### 全局注册表

```python
@register_component    # 装饰器自动注册
class CSVInputComponent(BaseComponent):
    ...

# 引擎通过名称查找:
comp = get_component("csv-input")
result = await comp.execute(config, inputs)
```

新增控件只需：继承基类 → 加 `@register_component` → 实现 `execute` 和 `get_config_schema` → 放到对应目录。前端自动从 `/api/components/built-in` 获取列表并渲染。

## 4. DAG 执行引擎

### 4.1 执行流程

```
作业上线
  │
  ▼
job_runner.start_job(job_id)
  │  后台 asyncio Task 启动
  ▼
从 MySQL 读取 dag_config
  │
  ▼
dag_executor.execute(dag_config)
  │
  ├── 1. Kahn 算法拓扑排序
  │
  ├── 2. 按序执行
  │     for node_id in sorted_ids:
  │       comp = get_component(name)
  │       upstream_data = outputs[prev_node]
  │       result = await comp.execute(config, upstream_data)
  │       outputs[node_id] = result
  │
  └── 3. 写 Execution 记录到 MySQL
```

### 4.2 横向扩展

```
                    ┌──────────────┐
                    │   MySQL 8     │
                    │ (作业配置)    │
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼─────┐      ┌────▼─────┐      ┌────▼─────┐
   │ Worker 1 │      │ Worker 2 │ ...  │ Worker N │
   │ Uvicorn  │      │ Uvicorn  │      │ Uvicorn  │
   │ :8080    │      │ :8081    │      │ :808N    │
   └──────────┘      └──────────┘      └──────────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
                    ┌──────▼──────┐
                    │   Redis 7   │
                    │ (任务队列)   │
                    └─────────────┘
```

- 每台机器启动一个 FastAPI 进程（含引擎）
- 通过 Redis 发布/订阅协调作业分配
- 多机器 = 自然横向扩展
- 或在单机上 `gunicorn -w 4` 启动多 Worker

### 5.2 核心表结构

```sql
-- 用户表
CREATE TABLE user (
    id         BIGINT AUTO_INCREMENT PRIMARY KEY,
    username   VARCHAR(50)  NOT NULL UNIQUE,
    password   VARCHAR(255) NOT NULL,
    email      VARCHAR(100),
    role       VARCHAR(20)  DEFAULT 'user',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 控件定义表
CREATE TABLE component (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    name          VARCHAR(100) NOT NULL UNIQUE,
    display_name  VARCHAR(100) NOT NULL,
    type          ENUM('input','process','output') NOT NULL,
    config_schema TEXT COMMENT 'JSON Schema',
    description   TEXT,
    icon          VARCHAR(50),
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 作业表
CREATE TABLE job (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    description TEXT,
    status      ENUM('draft','online','offline','error') DEFAULT 'draft',
    dag_config  TEXT COMMENT 'JSON: {nodes, edges}',
    created_by  BIGINT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES user(id)
);

-- 执行记录表
CREATE TABLE execution (
    id             BIGINT AUTO_INCREMENT PRIMARY KEY,
    job_id         BIGINT NOT NULL,
    status         ENUM('running','success','failed') DEFAULT 'running',
    rows_processed INT DEFAULT 0,
    started_at     DATETIME DEFAULT CURRENT_TIMESTAMP,
    finished_at    DATETIME,
    log            TEXT,
    error_msg      TEXT,
    FOREIGN KEY (job_id) REFERENCES job(id)
);
```

## 7. 启动 & 部署

### 本地开发

```bash
# 1. 启动 MySQL + Redis
docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=root mysql:8
docker run -d -p 6379:6379 redis:7

# 2. 创建数据库
mysql -uroot -proot -e "CREATE DATABASE IF NOT EXISTS bangsheng"

# 3. 启动后端（一行命令）
cd backend
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python run.py
# → http://localhost:8080
# → Swagger 文档: http://localhost:8080/docs

# 4. 启动前端
cd frontend
npm install
npm run dev
# → http://localhost:3000
```

#### 控件注册模式对比

```python
# Hamilton：用函数签名声明 DAG 节点
@extract_columns("A", "B")
def concat_fields(A: pd.Series, B: pd.Series) -> pd.Series:
    return (A + B).rename("C")

# 本平台：用装饰器注册控件类
@register_component
class ConcatComponent(BaseComponent):
    component_name = "concat"
    display_name = "字段拼接"

    async def execute(self, config, inputs):
        ...
```

两者设计理念一致，都是声明式注册 + 名字查找。
